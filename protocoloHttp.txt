

Perguntas e um exercício prático a serem resolvidos para discutirmos nesta aula:

1) Há diferença entre Internet e Web? Justifique.
A internet compõe todos os serviços que fazem da infraestrutura de redes. A Web é composta pela rede de páginas que se comunicam entre si, não necessariamente uma página web está disponível na internet, assim como algo que está na internet não necessariamente é uma página web, por exemplo um servidor, não necessariamente ele está disponível ou se comunicando com a internet e por estar não obrigatoriamente ele apresenta uma página web para receber quem encontra esse servidor.

2) Como funciona a Web? Que tipo de arquitetura esse serviço trabalha? Qual o papel do HTTP?
A Web funciona com base na internet, então uma página precisa estar conectada em uma rede para ser encontrada pelos computadores que acessam essa rede, um exemplo legal de pensar é um serviço de self host, onde todos os computadores de uma mesma rede conseguem encontrar uma determinada página mas não é possível encontrar essa mesma página de uma rede externa, a menos que ela seja disponibilizada com esse objetivo. O http é o protocolo de transferência de dados de hipertexto, ou seja uma página que tem como objetivo ser disponibilizada na web, seja ela aberta ou restrita a internet como um todo, precisa atender a esses requisitos para funcionar nesse modelo, existem outros modelos de protocolo que são utilizados amplamente, como o sftp, mais recentemente o protocolo amqp.

3) Explique o fluxo de comunicação que ocorre entre o navegador e o servidor Web.
A comunicação entre um servidor e o navegador web, de forma bem generica, funciona partindo de uma requisição feita pelo cliente, normalmnete um navegador web, mas podemos usar ferramentas como curl ou wget para fins de exemplo, essa requisição é encaminhada para o servidor que hospeada o dns da pagina, onde tem a tradução dos endereços nomeados para os endereços ips de onde o servidor essa hospedado de fato, esse servidor então encaminha a requisição do navegador(Cliente) com o caminho, por exemplo, o acesso a pagina http://www.exemplo.com/paginas/pagina_exemplo01.html vai procurar dentro do servidor que se encontra no o site www.exemplo.com/ a pasta paginas e dentro dessa pagina o documento pagina_exemplo01.html. O protocolo http nesse ponto se torna muito útil por ter formalizado algumas regras para as requisições, então sabemos que se caso a página não exista o erro que a requisição que fizemos para o servidor vai ser 404, caso a requisição aconteça com sucesso teremos um retorno 200, indicando o sucesso na nossa pesquisa.

4) Explique o protocolo HTTP (mensagens e informações do cabeçalho e do corpo).
O protocolo http, é divido em 3 partes, o header ou cabeçalho, onde normalmente constam informações sobre login, como tokens e dados mais sensíveis, o body ou corpo da mensagem, que pode conter um json com algumas informações que são do interesse do servidor, por exemplo em alguma requisição de api podemos usar o body para filtrar determinado período para uma consulta.

5) Quais são os métodos HTTP existentes? Qual a diferença entre os métodos GET e POST?
Os métodos que existem são o GET, POST, DELETE, PUT. O Método GET como o nome diz, é para consulta então ele recebe uma informação do servidor, por exemplo o ato de abrir uma página web é uma requisição de GET. o Método POST é método de envio de informações para o servidor, então iremos preencher o campo body da requisição com algumas informações que desejamos fornecer ao servidor e ele irá receber esse dado via método POST. O método DELETE é similar ao post mas iremos deletar uma informação filtrada conforme definição do servidor e o método POST que funciona como uma mistura do método GET com o método POST onde iremos atualizar um elemento específico, então iremos primeiro fazer um get para termos um elemento único e depois um POST para definição do campos de acordo com o definido no body da requisição.

6) Qual a diferença entre páginas estáticas e dinâmicas? Como trabalha o servidor Web estes tipos de páginas?
A página estática, não se atualiza durante o uso, ela pode conter um link para outra página mas se limita a isso, uma página dinâmica fica realizando requisições para o servidor que está para se comportar de acordo com a necessidade do usuário. Por exemplo uma página de formulário, onde o usuário digita o CEP e ele já preenche os dados de endereço deixando apenas alguns campos para o usuário preencher, é um exemplo de uma possivel página dinâmica, já que é necessário após o usuário preencher o CEP que essa informação seja enviada para o'servidor, que ele processo o dado primeiro verificando a validade da informação se atende aos requisitos e na sequência retorne com os dados desejados. O Servidor no caso de uma página dinâmica, está sempre comunicando com a página que está sendo fornecido, em caso de uma página estática, o servidor encaminha o dado para quem solicitou e a comunicação se encerra nesse ponto.

7) Onde entra o JavaScript no desenvolvimento Web? Para que serve e como é aplicado no funcionamento das aplicações Web?
Atualmente o Javascript se encontra tanto na parte do front end, quanto no back end, na parte do front  é amplamente utilizado para adicionar dinamicidade as páginas, seja através de alterações na página de acordo com a resolução da tela, seja por compreender que o usuário se tornou inativo e é inseguro manter o acesso ativo. Já na parte do backend o javascript se apresenta na comunicação com o servidor
